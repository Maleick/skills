# Memory Note Template

Use this structure for every durable note.

```markdown
---
title: "<component>: <decision or issue>"
date: "2026-02-18T00:00:00+00:00"
project: "<project>"
tags:
  - decision
  - troubleshooting
  - runbook
---

## Summary
- 1-3 bullets with the stable outcome

## Context
- What problem or goal triggered the work

## Decision
- Explicit, testable statement of what was chosen

## Rationale
- Why this option was selected over alternatives

## Implementation
- Paths, commands, and configuration details
- Never include secret values

## Verification
- Exact checks that confirm success

## Follow-ups
- Open questions, TODOs, or risks

## Changelog
- 2026-02-18T00:00:00+00:00: created via save_memory.
```

Tag guidance:
- Use 3-8 tags.
- Keep tags lowercase and hyphenated.
